## What the Template Creates

| Resource | Logical ID | Purpose |
|---|---|---|
| `AWS::SecretsManager::Secret` | `IAMTempPasswordSecret` | Auto-generated 16-char one-time password |
| `AWS::IAM::Group` | `S3UserGroup` | Grants `s3:ListAllMyBuckets` |
| `AWS::IAM::Group` | `EC2UserGroup` | Grants `ec2:DescribeInstances` (list only) |
| `AWS::IAM::ManagedPolicy` | `EC2CreateInstancePolicy` | Grants `ec2:RunInstances` (create) |
| `AWS::IAM::User` | `EC2User1` | Member of EC2UserGroup – **list only** |
| `AWS::IAM::User` | `EC2User2` | Member of EC2UserGroup + CreatePolicy – **list + create** |
| `AWS::IAM::User` | `S3User` | Member of S3UserGroup – **list S3 buckets only** |

### Permission Matrix

| User | List EC2 | Create EC2 | List S3 |
|---|:---:|:---:|:---:|
| ec2-user1 | ✅ | ❌ | ❌ |
| ec2-user2 | ✅ | ✅ | ❌ |
| s3-user   | ❌ | ❌ | ✅ |

All users:
- Have **console access** via a `LoginProfile`
- Share the **same temporary password** from Secrets Manager
- Are required to **change their password on first login** (`PasswordResetRequired: true`)

## Task 1 – Deploy via GitSync

### Prerequisites
- AWS Account with permissions to create IAM, CloudFormation, and Secrets Manager resources
- GitHub (or CodeCommit / GitLab / Bitbucket) repository
- AWS Console access

### Step 1 – Fork / Clone this repo
```bash
git clone https://github.com/<your-username>/bem11-iam-lab.git
cd bem11-iam-lab
```

### Step 2 – Create the CloudFormation Stack with GitSync

1. Open **AWS Console → CloudFormation → Stacks → Create Stack**
2. Choose **"With GitSync"**
3. Connect your Git provider and authorise AWS access
4. Select your repository, branch (`main`), and deployment file path (`deployment.yaml`)
5. Stack name: `bem11-iam-lab`
6. Click **Create**

CloudFormation will:
- Pull `iam-automation.yaml` from your repo
- Provision all resources
- Watch the branch for future commits and auto-update the stack

### Step 3 – Verify Deployment

```bash
# Check stack status
aws cloudformation describe-stacks \
  --stack-name bem11-iam-lab \
  --query "Stacks[0].StackStatus"

# Retrieve the temporary password
aws secretsmanager get-secret-value \
  --secret-id /iam/lab/temp-password \
  --query SecretString \
  --output text
```

Expected output (JSON):
```json
{"password":"Ab3$xKp9..."}
```

## GitSync Security Best Practices Applied

| Practice | Implementation |
|---|---|
| **Least privilege for GitSync role** | CloudFormation creates a scoped service-linked role |
| **Branch protection** | Enable required reviews on `main` branch in GitHub |
| **Secret rotation** | Secrets Manager secret can be rotated independently |
| **No hardcoded secrets** | Password generated by `GenerateSecretString`, never in template source |
| **Dynamic references** | `{{resolve:secretsmanager:...}}` keeps secrets out of CloudFormation console |
| **Resource tagging** | All resources tagged with `Project`, `ManagedBy` for governance |
| **`PasswordResetRequired: true`** | Temporary password is invalidated after first use |
